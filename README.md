# LeetCode Practice

NeetCode 150 — pattern-based study, Go solutions.

## Goal

Given any problem from the Blind 150: recognize the pattern in <5 min, produce a working solution in <30 min, explain time/space complexity, handle one follow-up.

## Schedule

### Phase 1 — Pattern Learning (Weeks 1–20)
- **1 problem/day, weekdays only** (5/week)
- Work patterns in order (see `progress.md`)
- Easy problems first, then Medium within each pattern
- 35-min hard cap on attempts — if stuck, read the *approach* (not code), then try again from scratch
- After every problem: fill out `notes.md` before moving on

### Phase 2 — Blind Solving (Weeks 21–26)
- **1 problem/day, weekdays only**
- Pick randomly across patterns — no pattern label
- Strict 30-min cap
- One mock interview per week: random problem, talk out loud, 45-min timer, log in `mocks/`

### Phase 3 — Maintenance (Ongoing after Week 26)
- **3 problems/week**, random, timed
- One mock per month
- Revisit any pattern that feels weak

## Structure

```
solutions/<pattern>/<NNN_problem_name>/
  solution.go   # your solution
  notes.md      # pattern, approach, complexity, gotchas
patterns/       # reference sheets per pattern (fill in as you go)
mocks/          # mock interview logs
progress.md     # master checklist
```
