LeetCode Study Plan

Alberto Quinonez  ·  Started June 2026  ·  NeetCode 150  ·  Python

Goal

Given any problem from the Blind 150: recognize the pattern in <5 min, produce a working solution in <30 min, explain time/space complexity clearly, and handle one follow-up.

Why 30 min, not 60? Real interviews are 45–50 min total. After intro and Q&A you have ~30–35 min for the actual problem. Training to 60 min trains you to be slow.

Three Phases

Phase 1
Pattern Learning
Weeks 1–20  ·  ~100 problems
  • 1 problem/day, weekdays only
  • Work patterns in order
  • Easy first, then Medium
  • 35-min hard cap — if stuck, read the approach (not code), then retry from scratch
  • Fill out notes.md every time
  • No timer pressure yet
Phase 2
Blind Solving
Weeks 21–26  ·  ~50 problems
  • 1 problem/day, weekdays only
  • Pick randomly — no pattern label
  • Strict 30-min cap
  • One mock interview per week
  • 45-min timer, talk out loud, no hints
  • Log every mock in mocks/
Phase 3
Maintenance
Week 27+  ·  ongoing
  • 3 problems/week, random, timed
  • One mock per month
  • Revisit weak patterns
  • Sustainable indefinitely
  • Ramp back to Phase 2 cadence when an interview is 4–6 weeks away

Daily Rhythm

Weekday Problem Session (~60 min)
  1. Read the problem, write out examples by hand (5 min)
  2. Identify the pattern — say it out loud or write it down before coding
  3. Code the solution, no looking anything up (30–35 min cap)
  4. If stuck past the cap: read the approach only, close it, re-implement from scratch
  5. Fill out notes.md — pattern name, key insight, complexity, gotchas
  6. Commit to git
Weekly Mock Interview (~45 min) — Phase 2 only
  1. Pick a random problem (use NeetCode shuffle or a friend)
  2. Start 45-min timer
  3. Talk through your approach before writing any code
  4. Write a working solution
  5. State time/space complexity unprompted
  6. After timer: fill out mock log in mocks/YYYY-MM-DD.md
  7. Self-grade 1–5 honestly

Rules of Thumb

Do
  • Write the pattern name before you write any code
  • State complexity unprompted — always, even in practice
  • Commit every solution, even ugly ones
  • Fill out notes.md while the problem is fresh
  • Revisit any problem you needed hints for after 3–5 days
  • Use NeetCode videos after your attempt, not before
Don't
  • Look at the solution before your 35-min cap expires
  • Skip notes.md when you solved it easily — those insights are still worth recording
  • Do multiple problems in one sitting to "catch up" — consistency beats bursts
  • Jump to Hard problems before the pattern is solid
  • Grind random problems — stick to the pattern order until Phase 2

Maintenance Mode (Phase 3)

The goal of maintenance is to keep pattern recognition sharp without making this a second job.
  • 3 problems/week on any schedule that fits — doesn't have to be weekdays
  • Always timed (30-min cap). No relaxed practice in maintenance mode
  • Rotate through all 18 patterns over time — don't get comfortable only doing your strong patterns
  • When an interview is 4–6 weeks out, ramp back to Phase 2 cadence (1/day + weekly mocks)
